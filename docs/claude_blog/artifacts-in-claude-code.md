# Claude Code 现已支持 Artifacts

> 原文：https://claude.com/blog/artifacts-in-claude-code<br/>
> 副标题：将进行中的工作预览为实时、可交互的网页——基于完整会话上下文构建，可与团队共享。<br/>
> 分类：Product announcements | 产品：Claude Code | 日期：2026年6月18日 | 阅读时长：5分钟

---

从今天起，Claude Code 可以将工作进展捕获为一个 artifact，把 Claude Code 的工作转化为实时、可共享的可视化页面——包括 PR 演示、系统说明、仪表盘和发布清单——并随着会话的推进自动更新。

一次 Claude Code 会话，可能是排查一起故障、重构某个服务，也可能是分析数月的数据。Artifacts 会把这些工作转化成任何人都能打开、探索的网页，比如一份可浏览的 PR 演示、一个可筛选排序的仪表盘，甚至是一份随着工作推进自动填写完成情况的发布清单。Artifacts 让团队协作变得更容易，团队可以把更多时间花在构建上，而不是同步状态。

## **基于你的会话上下文构建**

Claude Code 使用你会话的完整上下文——包括代码库、已连接的连接器，以及对话本身——来构建 artifact。一个故障排查页面可以汇聚代码中失败的测试及其背后的函数、来自已连接监控工具的错误激增数据，以及你刚刚完成的会话中得出的根因分析。使用 artifacts，你无需接入数据源或搭建基础设施——你只需提出需求，Claude Code 就会基于已有的一切为你构建页面。

## **原地实时更新的页面**

当 Claude Code 更新某个 artifact 时，打开的页面会原地刷新，团队成员在更新发布的那一刻就能看到。每次发布都是同一链接下的一个新版本，并保留版本历史，你可以随时恢复到任意版本；此外，一个图库页面可以让你浏览和管理所有已创建的 artifacts。

根据我们的内部测试，最常见的使用场景之一是调试。典型流程大致是这样：一位工程师在站会前启动一次故障调查。Claude Code 通读日志，发布一个 artifact：一份时间线、可疑的提交记录，以及错误率图表。她从页面顶部把链接分享给团队。等到站会开始时，Claude 已经随着调查的推进重新发布了两次，纳入了最新信息。有了 artifacts，团队成员和相关方不再需要"让我们过一遍智能体发现了什么"，因为所有人看到的是同一个页面、同样的上下文。

## **仅对你的组织私密可见**

每个 artifact 默认仅对其创建者私密可见。当你准备好后，可以直接从页面上把它分享给团队成员和你的组织。Artifacts 只对你所在组织的已认证成员可见，无法被设为公开。管理员可以通过组织级开关和基于角色的权限范围来管理访问权限、设置保留策略，并通过合规 API 获得组织范围内的可见性。

## **开始使用**

向你的会话请求一个 artifact——或者只需要求一些"可视化的东西"，*以下是按角色划分的一些创意：*

* **法务 / 开源合规**：对每个依赖项进行许可证审计，直接从代码仓库标记出 copyleft 协议。*"构建一个 artifact，列出每个第三方依赖及其许可证，标记出所有 copyleft 协议。"*
* **隐私**：绘制个人数据在代码中被采集、存储、记录位置的数据流向图。*"追踪我们在代码库中接触个人数据的所有位置，生成一个用于隐私审查的 artifact。"*
* **安全**：将发现的问题直接链接到具体代码行，让修复方案一目了然。*"根据这次审查的身份认证发现，构建一个 artifact，每一项都链接到对应代码。"*
* **FinOps / 平台财务**：从基础设施即代码中映射出云资源和成本驱动因素。*"从 Terraform 配置中映射我们的云资源，生成一个按服务分组、标出主要成本驱动因素的 artifact。"*
* **软件工程师**：一份评审者真正能看懂的 PR 或 bug 演示，从 diff 及其上下文代码中提取而成。"制作一个 artifact，演示这个 PR——包括 diff、设计思路，以及我做过的测试。"
* **设计师与前端工程师**：针对某个界面的多种 UX 方案，均基于你真实的组件构建，确保选中的方案可以直接上线。"给我一个 artifact，包含这个注册表单的 5 种 UX 变体，全部基于我们的组件库构建。"
* **资深工程师与架构师**：一张服务实际组织方式的关系图，基于真实的 import 依赖关系绘制，而非白板草图。"根据代码把支付服务的组织结构映射成一个 artifact。"
* **SRE 与值班工程师**：一个随着排查推进不断完善、最终成为事后复盘文档的故障页面。"把这次故障做成一个 artifact——时间线、可疑提交、来自监控的错误激增数据——并随着我的排查进展重新发布。"
* **工程经理**：一份基于已合并 PR 生成的、真实反映团队本周交付情况的页面。"根据 PR 构建一个 artifact，展示我团队本周合并了什么，按项目分组。"

Claude Code 会构建好页面并给你一个链接。你可以在浏览器或桌面应用中打开它，从页面顶部分享——后续更新会自动发布到同一个 URL。

## **可用范围**

Artifacts 目前面向 Claude Team 和 Enterprise 组织提供 Beta 版本，可在 Claude Code CLI 和桌面应用中使用，生成的页面可在任意浏览器中查看。
