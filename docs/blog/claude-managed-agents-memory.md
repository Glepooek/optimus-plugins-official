## 跨会话持续学习的智能体

托管智能体将生产级基础设施与专为性能调优的工具链相结合。记忆功能在此基础上进一步延伸：它针对需要跨会话持续改进、并能与其他智能体共享所学知识的长期运行智能体，在内部基准测试中进行了专项优化。

我们发现，当记忆建立在智能体已有工具之上时，效果最为显著。托管智能体上的记忆直接挂载到文件系统，因此 Claude 可以依赖同样的 bash 和代码执行能力——正是这些能力让它在智能体任务中表现出色。借助基于文件系统的记忆，[我们最新的模型](https://www.anthropic.com/news/claude-opus-4-7)能够保存更全面、更有条理的记忆，并在判断哪些内容值得为特定任务记录时更加审慎。

---

## 面向生产级智能体的可迁移记忆

记忆功能专为企业部署而设计，具备作用域权限、审计日志和完整的程序化控制能力。记忆存储库（Store）可在多个智能体之间共享，并支持设置不同的访问权限。例如，组织级别的存储库可以设为只读，而用户级存储库则允许读写操作。多个智能体可以同时访问同一个存储库，互不覆盖彼此的内容。

记忆以文件形式存在，可通过 API 导出和独立管理，开发者拥有完全的控制权。所有变更均有详细审计日志追踪，可以清晰地追溯到产生该记忆的具体智能体和会话。支持回滚至历史版本，或从历史记录中删除特定内容。更新内容也会以会话事件的形式呈现在 [Claude Console](https://platform.claude.com/) 中，开发者可以追踪智能体学到了什么、知识来源是哪里。

## 团队正在构建什么

各团队正在利用记忆功能来闭合反馈循环、加速验证流程，并替代定制化的检索基础设施：

- **Netflix** 的智能体可以跨会话携带上下文，包括需要多轮对话才能发现的洞察，以及人工在对话中途给出的纠正——而不再需要手动更新提示词和 skill。
- **乐天（Rakuten）** 的基于任务的长期运行智能体通过记忆功能从每次会话中学习，避免重蹈覆辙，将首次通过率的错误减少了 **97%**，整个过程在工作空间作用域和可观测性边界内完成。
- **Wisedocs** 在托管智能体上构建了他们的文档核验流水线，利用跨会话记忆识别和记录反复出现的文档问题，将核验速度提升了 **30%**。
- **Ando** 正在托管智能体上构建他们的职场消息平台，捕捉每个组织的交互方式，而无需自行搭建记忆基础设施。
