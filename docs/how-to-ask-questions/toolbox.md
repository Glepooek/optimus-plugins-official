# 提问工具箱

> 日常使用的速查手册，遇到问题时快速查阅。

---

## 提问前检查清单

每次提问前，花30秒过一遍这个清单：

```
□ 我的问题包含背景了吗？（Context）
□ 我的问题足够具体了吗？（Question）
□ 我说清楚想要什么程度的答案了吗？（Goal）
□ 我排除了模糊词了吗？（"这个"、"那个"、"一些"、"某种"）
□ 我提供了必要的环境/约束信息了吗？
```

**快速自检：** 如果你的问题超过30个字但不到80个字，大概率是好问题。太短说明信息不够，太长说明没聚焦。

---

## 问题生成器（5个启动器）

不知道问什么时，依次套用：

1. **是什么** — 这个东西的本质是什么？
2. **为什么** — 为什么要这样做？不这样做会怎样？
3. **怎么做** — 具体步骤是什么？
4. **什么条件** — 在什么情况下适用？边界在哪？
5. **如果…会怎样** — 改变某个条件，结果会变吗？

---

## 精炼四步法

问题太模糊时，逐步精炼：

```
模糊问题 → 加限定词 → 加具体例子 → 加对比 → 精确问题
```

**示例：**
1. "怎么优化性能？" → 
2. "怎么优化**数据库查询**性能？" → 
3. "怎么优化**百万级用户表的分页查询**性能？" → 
4. "**索引优化 vs 缓存 vs 读写分离**，哪个更适合我的场景？"

---

## 追问决策树

拿到答案后，用这个决策树决定下一步：

```
拿到答案后：
├─ 没完全理解 → 问"为什么"（原理追问）
├─ 想知道边界 → 问"如果"（条件追问）
├─ 想动手做 → 问"怎么"（执行追问）
├─ 想比较方案 → 问"还有什么替代方案"
└─ 想验证理解 → 问"我这样理解对不对：……"
```

## AI 对话专用技巧

| 技巧 | 示例 |
|---|---|
| 角色设定 | "假设你是一个有10年经验的后端工程师" |
| 输出格式 | "用表格对比"、"给出代码示例"、"分步骤列出" |
| 约束条件 | "用Python 3.11"、"不要用第三方库"、"控制在100行以内" |
| 迭代追问 | "这个方案的缺点是什么？"、"如果并发量增加10倍呢？" |

---

## 技术问题自查三步

提问前先做这三步：

1. **我试了什么？** — 排除已知方案
2. **错误信息是什么？** — 精确复制，不要概括
3. **最小复现是什么？** — 剥离无关因素

---

## 决策问题模板

```
背景：我在做[什么项目]，面临[什么选择]
选项：A方案[描述]，B方案[描述]
约束：[时间/成本/团队能力/风险]
目标：我希望[什么结果]
```
