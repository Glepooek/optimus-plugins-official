# 提问练习册

> 通过刻意练习内化提问能力。每个练习都有自检标准。

---

## 练习1：问题生成器练习

**目标：** 练习用5个启动器生成问题

**题目：** 面对以下主题，用5个启动器（是什么、为什么、怎么做、什么条件、如果…会怎样）各生成一个问题。

**主题：**
1. Docker 容器化
2. React 状态管理
3. 团队代码评审流程

**自检标准：**
- [ ] 每个主题生成了5个问题
- [ ] 问题之间没有重复
- [ ] 问题具体而非笼统

**参考答案（Docker）：**
- 是什么：容器和虚拟机的本质区别是什么？
- 为什么：不用Docker直接部署会有什么问题？
- 怎么做：怎么把一个Node.js项目容器化？
- 什么条件：什么情况下不适合用Docker？
- 如果…会怎样：如果容器内存超限会怎样？

## 练习2：精炼练习

**目标：** 练习把模糊问题精炼成好问题

**题目：** 把以下模糊问题精炼成好问题（至少经过3步精炼）。

1. "代码怎么优化？"
2. "数据库怎么设计？"
3. "怎么学编程？"

**自检标准：**
- [ ] 每步都在缩小范围
- [ ] 最终问题包含具体场景
- [ ] 最终问题有明确的选择或对比

**参考答案（"代码怎么优化？"）：**
1. "代码怎么优化？"（模糊）
2. "怎么优化**React组件的渲染性能**？"（加限定词）
3. "怎么优化**一个包含1000行数据的表格组件的渲染性能**？"（加具体例子）
4. "**虚拟滚动 vs 分页 vs 懒加载**，哪个更适合我的场景？"（加对比）

## 练习3：追问链练习

**目标：** 练习用三条追问链深入

**题目：** 给定一个初始问题和初步答案，用三条追问链各追问2层。

```
初始问题："什么是Redis？"
初步答案："Redis是一个内存数据库，常用于缓存。"
```

**自检标准：**
- [ ] 为什么链：追问了原理层面
- [ ] 如果链：追问了边界/异常情况
- [ ] 怎么链：追问了实操层面
- [ ] 每条链追问了2层（共6个追问）

**参考答案：**

**为什么链：**
1. "为什么Redis比MySQL快？" →
2. "为什么单线程模型在内存操作中反而比多线程更高效？"

**如果链：**
1. "如果Redis内存满了会怎样？" →
2. "如果设置了淘汰策略，但数据量突然暴增，Redis会卡住吗？"

**怎么链：**
1. "怎么在Node.js项目中使用Redis做缓存？" →
2. "怎么设置缓存过期时间，以及怎么处理缓存穿透？"

## 练习4：综合场景模拟

**目标：** 练习通过提问逐步澄清问题

**场景：** 同事说"帮我看看这个接口为什么慢"

**任务：** 通过提问，在5轮对话内定位到具体问题。

**提问框架：**
- 第1轮：问环境（什么接口？什么环境？）
- 第2轮：问现象（慢到什么程度？以前快吗？）
- 第3轮：问已知（你查了什么？发现什么？）
- 第4轮：问数据（数据量多大？并发多少？）
- 第5轮：问约束（能改表结构吗？能加缓存吗？）

**自检标准：**
- [ ] 5轮提问覆盖了环境、现象、已知、数据、约束
- [ ] 每轮问题都比上一轮更具体
- [ ] 最终能定位到一个可操作的方向

**示例对话：**

```
同事：帮我看看这个接口为什么慢
你：什么接口？在什么环境下慢？（环境）
同事：/api/users，生产环境
你：慢到什么程度？是一直慢还是最近才慢？（现象）
同事：响应要3秒，以前是200ms，最近一周变慢的
你：你查了什么？有什么发现吗？（已知）
同事：看了日志，没报错，就是慢
你：这个接口查的数据量大概多大？最近有数据增长吗？（数据）
同事：用户表有100万条，最近做了一次数据迁移，翻了一倍
你：能加索引吗？能加缓存吗？（约束）
同事：索引可以加，缓存暂时不行
→ 方向：检查索引情况，可能是数据量翻倍后索引失效或缺失
```

## 练习5：AI对话迭代练习

**目标：** 对比差问法和好问法的效果差异

**题目：** 同一个需求，分别用"差问法"和"好问法"与AI对话，对比结果差异。

**需求：** 实现一个用户注册功能

**差问法：** "帮我写个注册"

**好问法（用CQG）：**
> 背景：我在做一个React + Node.js的全栈项目，后端用Express，数据库用MySQL
> 问题：实现用户注册功能，需要邮箱验证和密码强度校验
> 目标：前端用React Hook Form，后端用bcrypt加密密码，包含错误处理和loading状态

**对比维度：**
- [ ] AI第一次回答的可用性（差问法 vs 好问法）
- [ ] 需要追问的次数
- [ ] 最终代码的完成度
- [ ] 代码是否包含错误处理

## 综合自评

完成所有练习后，回顾一下：

1. **问题生成器** — 我能快速生成问题了吗？
2. **精炼四步法** — 我能把模糊问题变具体了吗？
3. **追问链** — 我能深入追问了吗？
4. **综合场景** — 我能在实际对话中运用了吗？
5. **AI对话** — 我的提问质量提升了吗？

如果某个环节还不熟练，回去重做对应练习。
