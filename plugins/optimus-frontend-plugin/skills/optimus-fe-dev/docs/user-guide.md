# optimus-fe-dev 使用说明

> 给 AI **4 个链接**，它帮你写完前端页面 + 测试交付物。

---

## 准备材料

| 文档 | 格式 | 必填 |
|------|------|:----:|
| PRD 需求文档 | 飞书 / Confluence 链接 | Yes |
| 前端架构设计文档 | 飞书链接（没有可让 AI 生成） | Yes |
| 设计稿 | Figma / Sketch 链接 | Yes |
| API 接口文档 | Swagger / Markdown 链接 | Yes |
| 业务组件库 Storybook | URL（没有会自动探测） | -- |

---

## 30 秒上手

**打开 Claude Code，贴上 4 个链接即可：**

```text
帮我还原这个页面：
- PRD：https://xxx.feishu.cn/docx/PRD链接
- 架构文档：https://xxx.feishu.cn/docx/架构文档链接
- 设计稿：https://www.figma.com/design/设计稿链接
- 接口文档：https://xxx.feishu.cn/docx/接口文档链接
```

没有架构文档？先说一句：

```text
根据这个 PRD 生成前端架构设计文档：https://xxx.feishu.cn/docx/PRD链接
```

## 流程总览

```text
你贴链接 ──► AI 分析规划 ──► 你确认方案 ──► AI 写代码 ──► AI 出交付物 ──► 完成
  Step 1       Step 2        Step 3        Step 4        Step 5
```

**你只需要动手 2 次：** Step 1 贴链接，Step 3 确认方案。其余全自动。

### 每一步做了什么

| 步骤 | AI 做的事 | 你要做的 |
|:----:|----------|:--------:|
| 1 | 读取 4 份文档，校验完整性 | 贴链接 |
| 2 | 分析需求 / 设计稿 / 接口，探测技术栈，输出实施计划 | -- |
| 3 | -- | 确认方案 |
| 4 | 按依赖顺序生成代码（类型 → 接口 → 页面），每段代码 2 轮自动 review | -- |
| 5 | 输出 5 份测试交付物 + 全面质量验证 | -- |

### 5 份交付物

| 交付物 | 给谁 | 内容 |
|--------|:----:|------|
| 功能说明书 | 测试 / PM | 功能点、校验规则、状态枚举、权限 |
| 接口清单 | 测试 / 后端 | 请求参数、返回数据、错误场景 |
| UI 状态清单 | 测试 | 加载中 / 空数据 / 报错 / 正常等所有状态 |
| Mock 数据 | 前端 / 测试 | 各场景模拟数据（`src/mock/`） |
| 架构设计文档 | 前端团队 | 技术栈、目录、组件拆分、数据流 |

## 关于生成的代码

- **风格跟随项目** — 先读已有代码再生成，保证一致
- **组件优先复用** — 有现成组件不造轮子
- **设计稿 100% 还原** — 布局 / 间距 / 颜色 / 字体 / 圆角 / 阴影精确匹配
- **自动集成埋点** — PV / 点击 / 曝光 / 表单 / 错误全覆盖（神策 SDK）

**代码目录结构：**

```text
src/
├── main.tsx            # 入口 + 神策 SDK 初始化
├── styles/globals.css  # 全局样式
├── pages/              # 页面
├── components/         # 共享组件
├── api/                # 接口层
├── stores/             # 全局状态 (Zustand)
├── hooks/              # 数据 Hook (TanStack Query)
├── types/              # 类型定义
└── utils/              # 工具函数
```

**技术栈：** 已有项目自动探测适配。新项目推荐 React 18 + TS + Tailwind + Zustand + TanStack Query + Vite。

---

## 注意事项

- **4 份文档缺一不可** — 不齐不开工
- **设计稿别跳过** — 再复杂工具也能处理
- **确认方案别跳过** — 1 分钟确认省 1 小时返工
- **交付物自动生成** — 不用额外操心
- **技术栈不一样也没关系** — 工具适配你的项目
