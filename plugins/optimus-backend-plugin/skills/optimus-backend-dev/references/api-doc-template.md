### 1.2 认证方式

<!-- 描述接口认证方式，如 Bearer Token、API Key 等 -->

```
Authorization: Bearer <token>
```

### 1.3 请求格式

- Content-Type: `application/json`
- 字符编码: `UTF-8`

### 1.4 统一响应格式

**成功响应：**

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

**分页响应：**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [],
    "total": 100,
    "page": 1,
    "pageSize": 20
  }
}
```

**错误响应：**

```json
{
  "code": 10001,
  "message": "参数校验失败",
  "data": null
}
```

### 1.5 通用错误码

| 错误码 | 说明 | HTTP 状态码 |
|---|---|---|
| 0 | 成功 | 200 |
| 10001 | 参数校验失败 | 400 |
| 10002 | 未登录或 Token 过期 | 401 |
| 10003 | 无权限 | 403 |
| 10004 | 资源不存在 | 404 |
| 10005 | 请求频率超限 | 429 |
| 50000 | 服务器内部错误 | 500 |

### 1.6 通用请求参数

| 参数名 | 位置 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| Authorization | Header | string | 是 | 认证令牌（公开接口除外） |
| Content-Type | Header | string | 是 | 固定为 application/json |

### 1.7 分页参数约定

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---|---|---|
| page | int | 否 | 1 | 页码，从 1 开始 |
| pageSize | int | 否 | 20 | 每页数量，最大 100 |
