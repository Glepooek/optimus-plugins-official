## 5. API 设计概览 [必填]

### 5.1 RESTful 规范

<!-- 描述项目遵循的 API 设计规范 -->

- URL 命名：小写、连字符分隔（如 `/api/v1/user-profiles`）
- HTTP 方法语义：GET 查询、POST 创建、PUT 更新、DELETE 删除
- 状态码使用：200 成功、201 创建成功、400 参数错误、401 未认证、403 无权限、404 不存在、500 服务器错误

### 5.2 统一响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

### 5.3 分页规范

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [],
    "total": 100,
    "page": 1,
    "pageSize": 20
  }
}
```

### 5.4 错误码设计

| 错误码 | 说明 | HTTP 状态码 |
|---|---|---|
| 0 | 成功 | 200 |
| 10001 | 参数校验失败 | 400 |
| 10002 | 未登录 | 401 |
| 10003 | 无权限 | 403 |

> 详细接口文档见 `后端API接口文档`
