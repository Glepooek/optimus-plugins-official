# PRD 质量评估清单

## 评分说明
- 每维度满分 100 分，综合分取加权平均
- ✅ 完全满足：100分  ⚠️ 部分满足：60分  ❌ 不满足：0分

## 权重表
| 维度 | 权重 |
|------|------|
| 结构完整性 | 20% |
| 内容详细度 | 25% |
| 逻辑一致性 | 20% |
| 可执行性 | 20% |
| 规范性 | 10% |
| AI友好性 | 5% |

## 1. 结构完整性（20%）
- [ ] 包含第1章：文档信息（含版本历史）
- [ ] 包含第2章：需求概述（含背景/目标/价值/范围/关联）
- [ ] 包含第3章：用户分析（含画像/场景/痛点）
- [ ] 包含第4章：功能需求（含架构/流程/功能详细描述）
- [ ] 包含第5章：非功能需求（含埋点/性能/安全/兼容性）
- [ ] 包含第6章：验收标准（含核心指标/验收清单）
- [ ] 包含第7章：其他（含名词解释/原型链接）

## 2. 内容详细度（25%）
- [ ] 需求背景有数据支撑
- [ ] 需求目标量化（可衡量指标）
- [ ] 功能描述含优先级（P0/P1/P2）
- [ ] 功能描述含前置条件
- [ ] 功能描述含详细业务规则
- [ ] 功能描述含交互细节（步骤/UI说明）
- [ ] 功能描述含异常处理
- [ ] 埋点需求有表格（事件/参数/目的）
- [ ] 性能/安全要求有具体数值

## 3. 逻辑一致性（20%）
- [ ] 需求范围与功能需求章节一致（无遗漏/矛盾）
- [ ] 需求目标与验收标准对应
- [ ] 优先级划分合理（基���功能≥P1不可设P2）
- [ ] 与现有系统关联已说明
- [ ] 术语全文统一

## 4. 可执行性（20%）
- [ ] 前端开发可基于文档开展（有UI交互流程）
- [ ] 后端开发可基于文档开展（有业务规则/接口依据）
- [ ] 测试可基于文档编写用例（有验收标准/边界场景）
- [ ] 无需大量口头补充说明
- [ ] 异常场景已覆盖

## 5. 规范性（10%）
- [ ] 标题层级统一
- [ ] 并列内容使用列表
- [ ] 结构化数据使用表格
- [ ] 语言专业（无口语化表达）
- [ ] 术语统一（无混用）

## 6. AI友好性（5%）
- [ ] 结构化程度高（层级清晰）
- [ ] 上下文完整（背景+目标+规则）
- [ ] 无模糊表述（"适当"、"合理"等）
- [ ] 有具体示例
- [ ] 边界条件明确
