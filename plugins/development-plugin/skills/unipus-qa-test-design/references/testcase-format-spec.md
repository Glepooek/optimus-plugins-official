# 测试用例格式规范

## 层级说明

### H1 — 文档主标题
```markdown
# XX功能测试用例
```
整个文档唯一，描述被测功能范围。

### H2 — 功能模块
```markdown
## 用户登录功能
```
一个主要的功能模块或用户故事，名称前不使用数字序号。

### H3 — 功能测试点
```markdown
### 头像设置
### 昵称规则与唯一性校验
```
同一模块下不同测试关注点的分类，对应测试点文档中的"功能点"。

### H4 — 验证点
```markdown
#### 上传规则校验
#### 唯一性校验
#### 动态格式化显示
```
一个功能测试点下更细致的验证方面，对应测试点文档中的"验证类型"。

### H5 — 用例场景（核心单元）
```markdown
##### [REQ-AVATAR-001] 用户上传2MB大小的JPG格式头像文件[P0]
##### [REQ-REG-002] 用户尝试使用已注册邮箱 existing@example.com 进行注册[P1]
```
**最重要的层级**，每个 H5 代表一条独立的测试用例。

标题格式：`##### [需求ID] 用例场景描述[优先级]`

标题内容要求：
- **需求ID**放在最前面，用方括号包裹，多个ID用逗号分隔
- 包含关键动作（做什么）
- 包含关键条件（在什么状态下）
- 包含关键数据（用什么数据，如具体数值、格式、角色）
- **优先级**放在最后面，用方括号包裹，取值 `[P0]`、`[P1]`、`[P2]`
- **不加数字序号**
- 足够简洁，复杂流程拆分为多个 H5

优先级分级规则：
- **P0**：核心业务流程、安全认证、数据完整性、权限控制
- **P1**：重要功能验证、边界条件、关键异常处理、性能指标
- **P2**：UI细节、低优先级异常、动画效果、空状态展示

### H6 — 前置条件 / 测试步骤 / 预期结果

每个 H5 下固定包含测试步骤和预期结果两个 H6 节点，有前置依赖时在测试步骤前增加前置条件节点：

#### 前置条件（可选）
```markdown
###### 前置条件
- 1. 使用已报名课程的USER角色账号登录系统
- 2. 已进入口语练习列表页
```
- 仅在用例有明确前置依赖时添加，无前置条件时省略
- 常见前置条件类型：账号角色与登录状态、特定数据状态（如已完成N次练习）、特定页面入口（如已进入某页面）
- 描述简洁，一条前置条件一行

#### 测试步骤
```markdown
###### 测试步骤
- 1. 打开系统登录页面
- 2. 在邮箱输入框输入 existing@example.com
- 3. 在密码输入框输入正确密码
- 4. 点击"登录"按钮
```
- 步骤清晰完整，任何人仅凭步骤可独立执行
- 关键输入数据直接写在步骤中

#### 预期结果
```markdown
###### 预期结果
- 1. 登录操作成功
- 2.1 页面跳转至用户主页
- 2.2 顶部显示用户昵称
```
- 必须具体且可验证，不写"正常显示"等模糊描述
- 描述系统的状态变化、页面跳转、提示文案、数据变化
- 只描述**页面布局、元素名称、数据内容、系统行为**
- **禁止**包含具体 UI 样式细节：颜色值（如 #FFFFFF、rgba）、字号（如 14px）、字重（如 bold/500）、间距（如 margin/padding 数值）、圆角（如 border-radius）、字体名称（如 Inter）等 CSS 属性
- ✅ 正确：`卡片底部显示"开始练习"按钮，按钮为渐变色背景`
- ❌ 错误：`按钮背景为 linear-gradient(135deg, #A855F7 → #6366F1)，圆角14px，Inter Medium 14px，颜色 #FFFFFF`

## 完整结构示例

```markdown
## 用户注册功能
### 用户注册信息校验
#### 邮箱和密码规则校验
##### [REQ-REG-001] 用户使用有效的、未注册邮箱和符合要求的密码进行注册[P0]
###### 前置条件
- 1. 未登录状态，处于注册页面
###### 测试步骤
- 1. 在邮箱输入框输入未注册的有效邮箱 test@example.com
- 2. 在密码输入框输入符合要求的密码 Test@12345
- 3. 点击"提交注册"按钮
###### 预期结果
- 1. 注册成功
- 2.1 系统显示"注册成功，请检查邮箱完成账户激活"提示
- 2.2 数据库新增用户记录，状态为"待激活"

##### [REQ-REG-002] 用户尝试使用已注册邮箱 existing@example.com 进行注册[P1]
###### 前置条件
- 1. 未登录状态，处于注册页面
###### 测试步骤
- 1. 在邮箱输入框输入已注册邮箱 existing@example.com
- 2. 输入任意合法密码
- 3. 点击"提交注册"按钮
###### 预期结果
- 1. 注册操作失败
- 2. 系统提示"该邮箱已被注册，请尝试其他邮箱或直接登录"
- 3. 数据库用户记录总数不变
```

## XMind 兼容注意事项

- 严格遵守 H1~H6 层级，不跳级
- 无序列表项使用 `-` 开头
- 避免在标题中使用特殊 Markdown 符号（如 `|`、`>`）
- 每个 H6 下至少有一个列表项
