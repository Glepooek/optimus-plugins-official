# FR 拆解规则与映射关系

## 拆解粒度标准

| 规则 | 说明 | 示例 |
|------|------|------|
| 复合需求拆分 | 含"和"、"并且"的多操作需求拆成多条 | "支持上传和删除" → 两条 FR |
| 校验规则拆分 | 每个校验条件独立一条 FR | 文件大小校验、格式校验各一条 |
| 文件上传拆分 | 格式、大小、编码、时长分别独立 | MP4格式 FR-008、500M限制 FR-006、x264编码 FR-007 |
| 选项枚举列全 | 支持的所有选项在同一条 FR 中列出 | "支持中/英/法/德/日/西/阿七种语言" → 一条 FR |
| 原子边界 | 每条 FR 可被单独测试，不依赖其他 FR 的细节 | — |

## 编号规则

| 类型 | 格式 | 作用域 | 说明 |
|------|------|--------|------|
| 功能需求 | `FR-001` | 全文档连续 | 跨模块不重置 |
| 验收标准 | `AC-001` | 全文档连续 | 跨模块不重置 |
| 非功能需求 | `NFR-001` | 全文档连续 | 独立于 FR/AC 序列 |

## 常见 PRD 描述 → FR 映射

| PRD 描述类型 | 提取方式 | 典型示例 |
|-------------|---------|---------|
| 文件上传功能 | 格式/大小/编码/数量各独立一条 | "支持MP4格式" → `FR-008` |
| 校验规则 | 每个校验条件独立一条 | "检测视频是否包含人物" → `FR-010` |
| 选项/配置 | 列出完整选项范围 | "支持中/英/法/德/日/西/阿" → `FR-013` |
| 操作步骤 | 按步骤拆分，关键动作各一条 | "点击保存→仅应用当前页" → `FR-056` |
| 记忆/回显 | 独立一条 | "记忆上次选择" → `FR-020` |
| 通知功能 | 独立一条，注明通知方式 | "生成成功短信/邮件通知" → `FR-075` |
| 导出功能 | 每种格式独立一条 | 导出MP4 → `FR-077`，导出剪映工程 → `FR-079` |
| 状态展示 | 列出所有状态值 | "生成中、生成失败、已完成" → `FR-082` |
| 二次确认 | 独立一条 | "删除前二次确认" → `FR-091` |
| 第三方集成 | 独立一条，说明集成方式 | "接入SSO登录SDK" → `FR-121` |
| 权限控制 | 每类权限独立一条 | — |

## 注意事项

1. **忠于 PRD 原文**：不添加 PRD 未提及的功能，不改变需求含义
2. **标注本期范围**：PRD 中"本期不包含"的功能在 FR 描述末尾注明，如 `（本期不包含）`
3. **保留量化指标**：PRD 中的数值（如"500M以内"、"≥95%"）必须原样保留到 FR 或 AC
4. **关注隐含需求**：业务逻辑上必需但 PRD 未明确说明的功能（如登录状态持久化）也应提取为 FR
5. **区分功能与非功能**：性能、安全、兼容性归入 NFR 附录，不混入各模块 FR 列表
6. **面向测试人员**：避免产品术语或技术术语歧义，语言清晰易懂

## 子功能分组规则

- 同一业务场景下的 FR 归入同一子功能组
- 子功能组编号：`3.1`、`3.2`、`3.3`...
- 命名应反映该组 FR 的共同主题
- 每组 3-8 条 FR 为宜，过多则进一步拆分子组

## 非功能需求分类

| 分类 | 关注点 | 典型指标 |
|------|--------|---------|
| 性能指标 | 响应时间、并发数、处理速度 | 页面加载≤3秒，并发≥100用户 |
| 质量指标 | 准确率、自然度、分辨率 | 口型同步准确率≥95% |
| 兼容性要求 | 浏览器、操作系统、文件格式 | 支持Chrome/Edge/Safari/Firefox |
| 安全性要求 | 加密、权限、数据保护 | 文件加密存储、HTTPS传输 |
| 可用性要求 | 系统可用性、备份、容灾 | 可用性≥99.5%，故障恢复≤1小时 |
