# XMind 层级结构规范

## 飞书兼容的四级节点结构

飞书项目导入 XMind 测试用例时，识别以下固定四级结构：

```
L0  根节点（用例集）     — 导入时忽略，仅作容器
L1  用例标题             — 每个 L1 = 一条独立测试用例
L2  前置条件             — 用例的上下文信息
L3  测试步骤             — 具体操作步骤（可多个）
L4  预期结果             — 步骤对应的验证点（挂载在 L3 下）
```

## 各层级详细说明

### L0 — 根节点

- XMind 思维导图的中心主题
- 导入飞书时**不会被读取**，仅作为容器
- 建议填写文档标题，如 `XX项目测试用例`

### L1 — 用例标题

- 根节点的**直接子节点**
- 每个 L1 节点代表**一条独立的测试用例**
- 标题格式（含前缀和优先级）：`【项目名】【模块名】用例场景描述 [P0]`
- 标题格式（含前缀无优先级）：`【项目名】【模块名】用例场景描述`
- 标题格式（无前缀有优先级）：`用例场景描述 [P0]`
- 标题格式（无前缀无优先级）：`用例场景描述`
- 优先级标签始终追加在标题末尾，用方括号包裹

### L2 — 前置条件

- L1 的子节点，每条用例**仅一个** L2
- 内容格式：`模块 > 测试点 > 验证点 [需求ID: XXX]`
- 描述执行该用例前需要满足的条件和上下文

### L3 — 测试步骤

- L2 的子节点，可以有**多个** L3
- 每个 L3 节点代表一条操作步骤
- 步骤应清晰具体，包含操作对象和操作数据

### L4 — 预期结果

- L3 的子节点，每个步骤**最多一个** L4
- 描述执行该步骤后期望的系统行为
- 部分步骤可以没有 L4（中间过渡步骤）

## 步骤与预期的配对规则

采用**按索引一对一配对**策略：

```
步骤列表：[S1, S2, S3, S4, S5]
预期列表：[E1, E2, E3]

结果：
L3 S1 → L4 E1
L3 S2 → L4 E2
L3 S3 → L4 E3
L3 S4 → （无预期）
L3 S5 → （无预期）
```

当预期数量多于步骤时，多出的预期创建 `(验证)` 步骤节点承载：

```
步骤列表：[S1, S2]
预期列表：[E1, E2, E3, E4]

结果：
L3 S1       → L4 E1
L3 S2       → L4 E2
L3 (验证)   → L4 E3
L3 (验证)   → L4 E4
```

## XMind 文件结构

`.xmind` 文件本质是 ZIP 压缩包，包含：

```
*.xmind (ZIP)
├── content.json      — 思维导图节点数据（核心）
├── metadata.json     — 创建者信息
└── manifest.json     — 文件清单
```

### content.json 节点结构

```json
{
  "id": "唯一ID",
  "title": "节点标题",
  "children": {
    "attached": [
      { "id": "...", "title": "子节点", "children": { ... } }
    ]
  }
}
```

## 与六级 Markdown 的对应关系

| 六级 MD | 角色 | 四级 XMind | 转换方式 |
|---------|------|-----------|----------|
| H1 `#` | 文档标题 | L0 根节点 | 直接映射 |
| H2 `##` | 功能模块 | 合入 L2 | 路径第一段 |
| H3 `###` | 功能测试点 | 合入 L2 | 路径第二段 |
| H4 `####` | 验证点 | 合入 L2 | 路径第三段 |
| H5 `#####` | 用例场景 | L1 用例标题 | 提升为用例节点 |
| H6 优先级 | 优先级标签 | 合入 L1 尾部 | `[P0]` / `[P1]` |
| H6 需求ID | 追溯标识 | 合入 L2 尾部 | `[需求ID: xxx]` |
| H6 测试步骤 | 操作步骤 | L3 步骤节点 | 每条一个节点 |
| H6 预期结果 | 验证预期 | L4 预期节点 | 配对挂载 |
