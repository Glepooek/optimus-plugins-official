# 命名规范

## 请求命名

**格式**：`{序号}-{接口功能描述}`

| 示例 | 说明 |
|------|------|
| `1-登录接口` | 登录 |
| `2-查询用户列表` | GET 列表 |
| `3-新增用户` | POST 创建 |
| `4-修改用户` | PUT/PATCH 更新 |
| `5-删除用户` | DELETE |
| `5.1-验证修改后用户详情` | 更新后回查（子请求） |
| `6.1-验证删除后查询详情` | 删除后回查（子请求） |
| `7-提交任务(数字人解析)` | 枚举值请求，括号内写业务含义 |
| `8-提交任务(PPT解析)` | 同接口不同枚举值 |

**子请求规则**：回查请求序号格式为 `{原序号}.1`，如原请求为 `5-修改用户`，回查为 `5.1-验证修改后用户详情`。

## 组件命名

| 组件类型 | 命名格式 | 示例 |
|---------|---------|------|
| HeaderManager | `{接口名}Headers` | `登录接口Headers` |
| JSONPostProcessor（提取） | `提取{变量描述}` | `提取Token`、`提取用户ID` |
| ResponseAssertion（HTTP 200） | `{操作}{资源名}成功断言` | `保存背景图成功断言` |
| JSONPathAssertion（业务码） | `{操作}{资源名}业务码断言` | `更新背景图业务码断言` |
| ResponseAssertion（value 结果） | `{操作}{资源名}结果断言` | `删除背景图结果断言` |
| BeanShellAssertion（有效ID） | `验证{操作}返回有效ID` | `验证保存返回有效ID` |
| BeanShellAssertion（ID一致性） | `验证{资源名}ID一致性` | `验证用户ID一致性` |
| JSONPathAssertion（字段验证） | `验证详情{字段名}字段` | `验证详情名称字段` |
| JSONPathAssertion（更新验证） | `验证{字段名}已更新` | `验证名称已更新` |
| JSONPathAssertion（未改验证） | `验证{字段名}未被修改` | `验证URL未被修改` |

## 变量命名

| 变量用途 | 命名规范 | 示例 |
|---------|---------|------|
| 认证令牌 | `token` | 简洁，非 `authToken`、`accessToken` |
| 新创建资源ID | `new{资源名}Id` | `newUserId`、`newOrderId` |
| 已存在资源ID | `existing{资源名}Id` | `existingUserId` |
| 枚举值A产生的ID | `{枚举业务含义}TaskId` | `digitalHumanTaskId` |
| 枚举值B产生的ID | `{枚举业务含义}TaskId` | `pptTaskId` |
| 基础配置 | 全大写 | `BASE_URL`、`PORT` |

**枚举变量隔离原则**：同一接口不同枚举值产生的变量**必须使用不同变量名**，防止互相覆盖。
