# 命名规范和示例

## 文件命名格式

```
<版本>_<项目名称>_<文档类型>.<扩展名>
```

## 版本号规范

| 格式 | 适用场景 |
|---|---|
| `V1.0` | 正式首版 |
| `V2.0` | 大版本更新（架构调整、重大功能变更） |
| `V1.1`、`V1.2` | 小版本迭代（功能补充、细节修改） |
| `V0.1`、`V0.9` | 草稿、评审版本 |
| `Draft` | 初稿，待评审 |

## 文档类型参考

| 中文类型 | 说明 |
|---|---|
| 需求文档 | PRD、功能说明、业务需求 |
| 技术方案 | 系统设计、架构方案、技术选型 |
| 接口文档 | API 说明、接口规范 |
| 测试报告 | 功能测试、性能测试、验收报告 |
| 部署文档 | 部署说明、运维手册、发布记录 |
| 会议纪要 | 评审记录、讨论纪要 |
| 项目周报 | 进度报告、里程碑总结 |

## 完整示例

| 场景 | 子目录           | 文件名 |
|---|---------------|---|
| 技术共享中心，统一认证平台，技术方案 V2.0 | `GX_统一认证平台`   | `V2.0_统一认证平台_技术方案.md` |
| 高等教育事业部，英语教材平台，需求文档草稿 | `GJ_英语教材配套平台` | `Draft_英语教材配套平台_需求文档.docx` |
| 基础教育事业部，K12 题库系统，测试报告 V1.0 | `JC_K12题库系统`  | `V1.0_K12题库系统_测试报告.pdf` |
| 语言服务中心，翻译平台，接口文档 V1.2 | `YF_翻译平台`     | `V1.2_翻译平台_接口文档.md` |

## 注意事项

- 项目名称保持一致（不同版本文档使用相同项目名称）
- 避免使用空格，可用下划线 `_` 分隔（下划线已作分隔符，不重复使用）
- 中文命名优先，英文缩写需全员统一认知后使用
- 扩展名按实际文件格式填写（`.md`、`.docx`、`.pdf`、`.xlsx` 等）
