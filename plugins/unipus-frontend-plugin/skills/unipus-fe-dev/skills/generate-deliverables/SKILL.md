# 阶段 4：生成测试交付物

代码生成完成后，生成面向测试团队的标准化交付物。

**宣告：** "我正在生成测试交付物。"

## 交付物清单

| # | 交付物 | 说明 |
|---|--------|------|
| 1 | 页面功能说明书 | 功能清单、表单规则、状态枚举、权限 |
| 2 | 接口清单与映射表 | 接口总览、请求/响应示例、异常场景 |
| 3 | 页面状态清单 | 所有可能出现的 UI 状态 |
| 4 | Mock 数据 | 正常/空/异常场景的 Mock |
| 5 | 前端架构设计文档 | 技术栈、目录结构、组件拆分、数据流 |

## 流程

### Step 1：收集上下文

- 读取本次任务生成的所有文件（页面、组件、API、类型定义）
- 读取关联的 PRD / 接口文档 / 设计稿信息
- 读取项目已有的路由配置，确认页面路径

### Step 2：生成页面功能说明书

按照 AI-DEV-STANDARDS.md 第七章 7.1 的规范输出：

- **基本信息**：页面路径、路由、关联接口、关联需求
- **功能清单**：序号、功能点、交互说明、对应组件/接口
- **表单验证规则**：字段、规则、提示信息
- **状态说明**：状态枚举值及含义
- **权限控制**：操作、所需权限、无权限表现

### Step 3：生成接口清单与映射表

按照 AI-DEV-STANDARDS.md 第七章 7.2 的规范输出：

- **接口总览**：接口名称、方法、路径、前端函数、文件位置
- **请求参数示例**：每个接口的请求 JSON 示例
- **预期响应示例**：每个接口的响应 JSON 示例
- **异常场景清单**：错误码场景、模拟方式、前端预期行为（401/403/404/422/500/超时/重复提交）

### Step 4：生成页面状态清单

列出页面所有可能出现的 UI 状态，供测试覆盖：

- **页面状态**：正常加载、空数据、加载中、加载失败、网络断开
- **表单状态**：新增模式、编辑模式、验证失败、提交中、提交成功、提交失败
- **交互状态**：按钮 hover/active/disabled、表格行高亮、分页切换、搜索重置

### Step 5：生成 Mock 数据

- 正常数据场景的 Mock
- 空数据场景的 Mock
- 错误场景的 Mock
- 放在项目的 `src/mock/` 目录下（或项目已有的 Mock 目录）

### Step 6：生成前端架构设计文档

输出本次开发涉及的前端架构设计说明：

- **项目技术栈**：框架、UI 组件库、状态管理、构建工具版本信息
- **目录结构说明**：本次新增/修改的目录及各目录职责
- **页面 & 路由设计**：页面清单、路由路径、路由参数、嵌套关系
- **组件拆分**：页面级组件与业务组件的拆分依据、组件树层级关系
- **数据流设计**：状态管理方案、跨组件通信方式
- **接口对接方案**：API 模块划分、统一错误处理
- **类型体系**：核心类型定义文件位置、类型复用关系
- **权限方案**：路由级权限、按钮级权限
- **关键设计决策**：重要技术选型或方案取舍及其理由

### Step 7：生成组件 API 文档（如涉及组件开发）

按照 AI-DEV-STANDARDS.md 第七章 7.3 的规范输出：

- Props 定义（属性、类型、默认值、必填、说明）
- Events 定义（事件名、参数、说明）
- Slots 定义（名称、参数、说明）
- 测试要点
