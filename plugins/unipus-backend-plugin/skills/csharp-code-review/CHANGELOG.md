# Changelog

## [1.4.0] - 2026-07-08

### Added
- fallback 表新增一行：修正代码用到 C# 9+ 新语法时须注明所需最低版本，无法确认用户项目 langVersion 时提供兼容旧版本的替代写法（此前完全没有覆盖此风险，若用户项目是 .NET 6 及更早，照搬修正代码会编译不通过）

## [1.3.0] - 2026-07-08

### Added
- 新增「⛔ 不要做什么（反例黑名单）」章节，6 条反模式（编造违规凑数、只查风格忽视健壮性、severity 凭直觉判定、臆测不完整代码、省略修正代码、过度严格化）

## [1.2.0] - 2026-07-08

### Added
- 报告格式后新增「触发条件/一线处理/仍失败兜底」fallback 表，覆盖未提供代码、代码不完整、非 C# 代码、零违规、严重级别判定歧义五种边界场景
- 明确严重级别判定原则：会导致 bug/崩溃/安全漏洞 → 🔴，仅影响可维护性/风格 → 🟡（解决实测中"公共字段未封装"判级不一致的问题）

## [1.1.0] - 2026-07-08

### Added
- 新增第 12 类审查类别「API 设计与健壮性」：异常处理语义统一、取消令牌传播、隐式依赖契约说明、异常不能被静默吞没
- 速查表补充异常吞没、缺少取消令牌两条常见违规
- 快速审查清单补充健壮性检查项

## [1.0.0] - 2026-07-08

### Added
- 初始版本纳入版本管理（此前 SKILL.md 缺少 `version` frontmatter 字段和 CHANGELOG.md）
