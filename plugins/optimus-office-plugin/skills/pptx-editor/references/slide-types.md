# 幻灯片页面类型

将**每张幻灯片**归类为以下 5 种类型之一：

## 1. 封面页

- **用于**：开场 + 定调
- **内容**：大标题、副标题/演讲者、日期/场合、强烈的背景/主题图形

### 布局选项

**左右不对称布局**
- 文字集中于一侧，图片在对面
- 适合：企业演示、产品发布、专业报告
```
|  标题与副标题  |    视觉/图片    |
|  描述文字     |               |
```

**居中对齐布局**
- 内容居中，带背景图片
- 适合：励志演讲、活动演示、创意提案
```
|                                        |
|           [背景图片]                   |
|              主标题                    |
|              副标题                    |
|                                        |
```

### 字号层级

| 元素 | 推荐大小 | 与基准比值 |
|---------|-----------------|---------------|
| 主标题 | 72-120px | 3x-5x |
| 副标题 | 28-40px | 1.5x-2x |
| 辅助文字 | 18-24px | 1x（基准） |
| 元信息（日期、姓名） | 14-18px | 0.7x-1x |

**关键原则：**
1. **强烈对比**：主标题应至少是副标题的 2-3 倍大
2. **视觉锚点**：最大的文字成为视觉焦点
3. **清晰层级**：观众应能立即判断哪个最重要
4. **避免相似**：相邻文字元素的字号差距不应小于 20%

### 内容元素

1. **主标题** — 必须包含，字号最大
2. **副标题** — 需要补充上下文时使用（明显小于标题）
3. **图标** — 能强化主题时使用
4. **日期/活动信息** — 相关时使用（字号最小）
5. **公司/品牌 Logo** — 代表组织时使用
6. **演讲者姓名** — 主旨演讲时使用（小号，低调）

### 设计决策

考量：目的（企业/教育/创意）、受众、基调、内容量、所需视觉素材。

### 工作流

1. **分析**：理解主题、受众、目的
2. **选择布局**：根据内容选择
3. **编写幻灯片**：使用 PptxGenJS，用形状和 SVG 元素增添视觉趣味
4. **验证**：生成预览 `slide-XX-preview.pptx`，用 `python -m markitdown slide-XX-preview.pptx` 提取文字，确认所有内容存在且无占位文字残留

## 2. 目录页

- **用于**：导航 + 预设期望（3-5 章节）
- **内容**：章节列表（可选图标/页码）

### 布局选项

**编号竖向列表** — 适合 3-5 章节，直观的演示文稿
```
|  目录                         |
|                               |
|  01  第一章标题               |
|  02  第二章标题               |
|  03  第三章标题               |
```

**双列网格** — 适合 4-6 章节，内容丰富的演示文稿
```
|  目录                           |
|                                  |
|  01  第一章   02  第二章         |
|      描述         描述           |
|  03  第三章   04  第四章         |
```

**侧边栏导航** — 适合 3-5 章节，现代/企业风格
```
| ▌01 |  第一章标题              |
| ▌02 |  第二章标题              |
| ▌03 |  第三章标题              |
```

**卡片式** — 适合 3-4 章节，创意/现代风格
```
|  目录                                 |
|  ┌─────┐  ┌─────┐  ┌─────┐  ┌─────┐  |
|  │ 01  │  │ 02  │  │ 03  │  │ 04  │  |
|  │标题 │  │标题 │  │标题 │  │标题 │  |
|  └─────┘  └─────┘  └─────┘  └─────┘  |
```

### 字号层级

| 元素 | 推荐大小 | 与基准比值 |
|---------|-----------------|---------------|
| 页面标题（"目录"/"议程"） | 36-44px | 2.5x-3x |
| 章节编号 | 28-36px | 2x-2.5x |
| 章节标题 | 20-28px | 1.5x-2x |
| 章节描述 | 14-16px | 1x（基准） |

**关键原则：**
1. **清晰编号**：章节编号应视觉突出——粗体、强调色或更大字号
2. **可扫视结构**：观众应在 2-3 秒内扫完所有章节
3. **一致间距**：章节间垂直间距相等
4. **视觉标记**：用彩色点、线条、数字或图标锚定每个章节
5. **避免杂乱**：描述文字一行以内，或干脆不写

### 内容元素

1. **页面标题** — 必须包含（"目录"、"议程"、"概览"）
2. **章节编号** — 格式一致（01、02... 或 I、II...）
3. **章节标题** — 简洁明了
4. **章节描述** — 可选的一行摘要
5. **视觉分隔** — SVG 分割线或间距
6. **装饰元素** — 低调的强调形状
7. **页码徽标** — **必须包含**

### 设计决策

1. **章节数量**：3 → 竖向列表；4-6 → 网格或紧凑；7+ → 多列
2. **描述长度**：长 → 竖向列表；无 → 紧凑网格/卡片
3. **基调**：企业 → 编号列表；创意 → 卡片式；学术 → 罗马数字
4. **一致性**：视觉风格与封面页保持一致

### 工作流

1. **分析**：章节列表、数量、演示文稿背景
2. **选择布局**：根据章节数量和内容选择
3. **规划视觉层级**：编号风格、字号、间距
4. **编写幻灯片**：使用 PptxGenJS，用形状做装饰元素。**必须包含页码徽标。**
5. **验证**：生成预览，用 markitdown 提取文字，确认内容和徽标存在。

## 3. 分节页

- **用于**：主要部分间的明确过渡
- **内容**：章节编号 + 标题（+ 可选 1-2 行介绍）

### 布局选项

**粗体居中** — 适合极简、现代风格演示文稿
```
|                  02                    |
|           章节标题                     |
|         可选介绍文字                   |
```

**左对齐加强调块** — 适合企业、结构化演示文稿
```
| ████ |  02                            |
| ████ |  章节标题                      |
| ████ |  可选介绍文字                  |
```

**分割背景** — 适合高对比度、戏剧性过渡
```
| ██████████ |     章节标题              |
| ██  02  ██ |     可选介绍              |
| ██████████ |                          |
```

**全出血背景加遮罩** — 适合创意、大胆风格演示文稿
```
| ████████████████████████████████████  |
| ████     大号 02        █████████    |
| ████  章节标题          █████████    |
| ████████████████████████████████████  |
```

### 字号层级

| 元素 | 推荐大小 | 说明 |
|---------|-----------------|-------|
| 章节编号 | 72-120px | 粗体，强调色或半透明 |
| 章节标题 | 36-48px | 粗体，清晰，主文字色 |
| 介绍文字 | 16-20px | 轻字重，低调颜色，可选 |

**关键原则：**
1. **戏剧性数字**：章节编号 = 最突出的视觉元素
2. **强烈标题**：大号但明显次于编号
3. **内容极简**：只有编号 + 标题 + 可选一行文字
4. **充足留白**：留出大量空白——分节页是停顿时刻

### 内容元素

1. **章节编号** — 必须包含。格式：`01`、`02`... 或 `I`、`II`...。与目录页风格一致。
2. **章节标题** — 必须包含，简洁明了。
3. **介绍文字** — 可选 1-2 行描述。
4. **装饰元素** — SVG 强调形状（条、线、几何块）。
5. **页码徽标** — **必须包含**。

### 设计决策

1. **基调**：企业 → 强调块；创意 → 全出血；极简 → 粗体居中
2. **颜色**：背景/强调用强烈的调色板颜色；高对比度文字
3. **一致性**：整个演示文稿中所有分节页使用相同风格
4. **与内容页形成对比**：视觉上要明显不同（不同背景色、更多留白）

### 工作流

1. **分析**：章节编号、标题、可选介绍
2. **选择布局**：根据内容和基调选择
3. **编写幻灯片**：使用 PptxGenJS，用形状做装饰元素。**必须包含页码徽标。**
4. **验证**：生成预览，提取文字，确认内容和徽标存在。

## 4. 内容页

根据内容选择子类型。每张内容页属于且仅属于一种子类型：

### 子类型

**文字型** — 要点、引言或短段落
- 仍然需要图标或 SVG 形状——绝不纯文字
```
|  幻灯片标题                          |
|  * 要点一                            |
|  * 要点二                            |
|  * 要点三                            |
```

**混合媒体型** — 双列或半出血图片+文字
```
|  幻灯片标题                          |
|  文字内容     |  [图片/视觉元素]   |
|  和要点       |                   |
```

**数据可视化型** — 图表（SVG 柱/进度/环形）+ 要点
- 必须包含数据来源
```
|  幻灯片标题                          |
|  [SVG图表]    |  关键发现 1         |
|              |  关键发现 2         |
|              来源：xxx              |
```

**对比型** — 并排列或卡片（A vs B，优缺点）
```
|  幻灯片标题                          |
|  ┌─ 选项A ─┐  ┌─ 选项B ─┐           |
|  │  细节1  │  │  细节1  │           |
|  └─────────┘  └─────────┘           |
```

**时间线/流程型** — 带箭头的步骤、旅程、阶段
```
|  幻灯片标题                          |
|  [1] ──→ [2] ──→ [3] ──→ [4]        |
|  步骤    步骤    步骤    步骤         |
```

**图片展示型** — 主图、画廊、视觉优先布局
```
|  幻灯片标题                          |
|  ┌────────────────────────────────┐  |
|  │         [主图]                 │  |
|  └────────────────────────────────┘  |
|  说明文字或补充文字                  |
```

### 字号层级

| 元素 | 推荐大小 | 说明 |
|---------|-----------------|-------|
| 幻灯片标题 | 36-44px | 粗体，幻灯片顶部 |
| 章节标题 | 20-24px | 粗体，用于幻灯片内子章节 |
| 正文 | 14-16px | 常规字重，左对齐 |
| 说明文字/来源 | 10-12px | 低调颜色，最小文字 |
| 数据展示 | 60-72px | 大号粗体数字，用于关键统计 |

**关键原则：**
1. **正文左对齐** — 绝不居中对齐段落或列表
2. **字号对比** — 标题必须 36pt+ 才能从 14-16pt 正文中突出
3. **视觉元素必须** — 每张内容页至少有一个非文字元素
4. **留白** — 最小 0.5" 边距，内容块间 0.3-0.5"

### 内容元素

1. **幻灯片标题** — 必须包含，位于幻灯片顶部
2. **正文内容** — 根据子类型：文字、要点、数据或对比
3. **视觉元素** — 图片、图表、图标或 SVG 形状——必须包含
4. **来源/说明** — 展示数据或外部内容时使用
5. **页码徽标** — **必须包含**

### 设计决策

1. **子类型**：首先确定——决定整体布局
2. **内容密度**：密集 → 多列或更小字号；稀疏 → 更大元素加更多留白
3. **数据 vs 叙事**：数据密集 → 图表+统计展示；故事导向 → 图片+引言
4. **多样性**：每张内容页应与前一张使用不同布局
5. **一致性**：字体、颜色和间距必须与演示文稿其余部分一致

### 工作流

1. **分析**：内容，确定子类型，规划布局
2. **选择布局**：最适合子类型和内容量的布局
3. **编写幻灯片**：使用 PptxGenJS，用形状做图表、装饰元素、图标。**必须包含页码徽标。**
4. **验证**：生成预览 `slide-XX-preview.pptx`，用 markitdown 提取文字，确认所有内容存在、无占位文字、包含徽标。

## 5. 总结/结尾页

- **用于**：收尾 + 行动引导
- **内容**：关键要点、CTA/下一步、联系方式/二维码、感谢

### 布局选项

**关键要点** — 适合教育、企业、数据驱动演示文稿
```
|  关键要点                            |
|  ✓  要点一                           |
|  ✓  要点二                           |
|  ✓  要点三                           |
```

**CTA / 下一步** — 适合销售提案、方案展示、项目启动
```
|  下一步                              |
|  [1] 行动一                          |
|  [2] 行动二                          |
|  联系方式：email@example.com         |
```

**感谢/联系方式** — 适合大会演讲、主旨发言
```
|            感谢                       |
|         name@company.com             |
|         @handle | website.com       |
```

**分栏回顾** — 适合既需要回顾又需要行动的演示文稿
```
|  总结            |  下一步           |
|  * 要点一        |  联系我们：       |
|  * 要点二        |  email@co.com     |
|  * 要点三        |  [二维码]         |
```

### 字号层级

| 元素 | 推荐大小 | 说明 |
|---------|-----------------|-------|
| 结尾标题（"感谢"/"总结"） | 48-72px | 粗体，有力度 |
| 要点/行动项 | 18-24px | 清晰，可扫视 |
| 辅助文字 | 14-16px | 常规字重 |
| 联系信息 | 14-16px | 低调颜色 |

**关键原则：**
1. **有力的结束语**：主要信息应最大最突出
2. **可扫视的条目**：要点/行动项简洁（每条一行）
3. **联系方式清晰**：易读但不过于显眼
4. **令人印象深刻的结尾**：自信、精致地收场

### 内容元素

1. **结尾标题** — 必须包含
2. **要点** — 3-5 个简洁总结（如适用）
3. **行动号召** — 清晰的下一步（如适用）
4. **联系信息** — 邮箱、网站、社交账号（如提供）
5. **装饰元素** — SVG 强调形状，保持视觉一致性
6. **页码徽标** — **必须包含**

### 设计决策

1. **结尾类型**：回顾、CTA、感谢，或组合
2. **内容量**：多个要点 → 列表；简单结尾 → 居中感谢
3. **受众行动**：受众需要做某件事 → CTA；信息分享 → 要点总结
4. **基调一致性**：与封面页的能量匹配
5. **视觉区分**：特别但不脱离整体风格

### 工作流

1. **分析**：结尾内容——要点、CTA、联系方式、感谢
2. **选择布局**：根据内容类型选择
3. **编写幻灯片**：使用 PptxGenJS，用形状做装饰元素。**必须包含页码徽标。**
4. **验证**：生成预览，提取文字，确认内容和徽标存在。

## 附加布局模式

在内容页间使用这些布局增加视觉多样性：

- **双列**（左文字右插图）
- **图标+文字行**（图标置于彩色圆形中，粗体标题，下方描述）
- **2x2 或 2x3 网格**（一侧图片，另一侧内容块网格）
- **半出血图片**（完整占据左侧或右侧）加内容叠加
- **大数据展示**（60-72pt 大号数字，下方小标签）
- **对比列**（前后对比、优缺点）
- **时间线或流程图**（编号步骤、箭头）
- **图标置于小彩色圆形中**位于章节标题旁
- **斜体强调文字**用于关键统计数据或标语
