## 测试点模板

### 基础模板
```markdown
- TP_[模块]_[功能]_[序号]: [测试点名称]
  - 验证要点: [具体需要验证什么]
  - 优先级: 高/中/低
  - 关联需求: REQ_XXX_XXX_XXX
  - 派生测试用例: X个
```

### 集成验证点模板（含依赖）
```markdown
- TP_[模块]_[功能]_[序号]: [测试点名称]
  - 验证要点: [具体需要验证什么]
  - 优先级: 高/中/低
  - 关联需求: REQ_XXX_XXX_XXX, REQ_XXX_XXX_XXX
  - 派生测试用例: X个
  - 依赖关系: 依赖 TP_XXX_XXX_XXX
```

## 测试点示例

### 功能验证点示例
```markdown
- TP_USER_LOGIN_001: 用户正常登录
  - 验证要点: 验证用户使用正确用户名密码可成功登录系统
  - 优先级: 高
  - 关联需求: REQ_USER_FUNC_001
  - 派生测试用例: 2个
```

### 边界验证点示例
```markdown
- TP_USER_LOGIN_002: 用户名长度限制验证
  - 验证要点: 验证用户名长度在1-20字符范围内的处理，含边界值和超界值
  - 优先级: 中
  - 关联需求: REQ_USER_DATA_001
  - 派生测试用例: 3个
```

### 异常验证点示例
```markdown
- TP_USER_LOGIN_003: 错误密码处理
  - 验证要点: 验证输入错误密码时的错误提示和连续失败后的账户锁定机制
  - 优先级: 高
  - 关联需求: REQ_USER_FUNC_001, REQ_USER_SEC_001
  - 派生测试用例: 2个
```

### 集成验证点示例
```markdown
- TP_USER_LOGIN_004: 登录后权限加载验证
  - 验证要点: 验证登录成功后用户权限正确加载并在各功能模块生效
  - 优先级: 高
  - 关联需求: REQ_USER_FUNC_001, REQ_USER_INTG_001
  - 派生测试用例: 2个
  - 依赖关系: 依赖 TP_USER_LOGIN_001
```

## 需求追溯矩阵模板

### 正向追溯（需求 → 测试点 → 测试用例）
```
需求 REQ_USER_FUNC_001
  ↓ 派生
  测试点 TP_USER_LOGIN_001
  测试点 TP_USER_LOGIN_002
    ↓ 派生
    测试用例 TC_USER_LOGIN_001
    测试用例 TC_USER_LOGIN_002
```

### 反向追溯（测试用例 → 测试点 → 需求）
```
测试用例 TC_USER_LOGIN_001
  ↑ 来源于
测试点 TP_USER_LOGIN_001
  ↑ 验证
需求 REQ_USER_FUNC_001
```
