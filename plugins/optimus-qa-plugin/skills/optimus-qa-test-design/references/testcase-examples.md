# 测试用例转换示例

## 测试点 → 测试用例完整示例

### 输入：测试点文档片段

```
#### 头像设置功能

##### 功能验证点
- 头像上传功能
  - 验证要点: 验证用户能够成功上传 JPG/PNG 格式的头像图片
- 头像自动压缩功能
  - 验证要点: 验证上传的头像能够自动压缩至 200x200 像素

##### 边界验证点
- 头像文件大小限制
  - 验证要点: 验证头像文件大小 ≤2MB 的限制处理

##### 异常验证点
- 不支持格式文件上传处理
  - 验证要点: 验证上传非 JPG/PNG 格式文件时的错误提示
```

## 边界验证点拆分规则

一个边界验证点通常需要拆分为以下场景：

| 场景 | 说明 | 示例（限制≤20字符） |
|------|------|---------------------|
| 最小值 | 下边界合法值 | 输入 1 个字符 |
| 最大值 | 上边界合法值 | 输入 20 个字符 |
| 略小于下边界 | 非法值（如空） | 输入 0 个字符 |
| 略大于上边界 | 非法值 | 输入 21 个字符 |

---

## 异常验证点用例设计要点

- 明确写出触发异常的具体操作和数据
- 预期结果必须包含：操作结果（失败/拒绝）+ 提示文案（具体内容）+ 系统状态（数据未变化/回滚等）
- 不同异常原因各写一条用例（如：格式错误、大小超限、网络异常分别独立）
